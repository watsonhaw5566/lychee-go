# cron · 定时任务

支持标准 6 字段 Cron 表达式的定时任务调度器。

## Cron 表达式

```
*  *  *  *  *  *
│  │  │  │  │  │
│  │  │  │  │  └── 星期 (0-6, 0=周日)
│  │  │  │  └────── 月份 (1-12)
│  │  │  └────────── 日期 (1-31)
│  │  └────────────── 小时 (0-23)
│  └────────────────── 分钟 (0-59)
└────────────────────── 秒 (0-59)  ← 比标准 Linux Cron 多了"秒"字段
```

## 常用示例

| 表达式 | 含义 |
|--------|------|
| `*/5 * * * * *` | 每 5 秒 |
| `0 * * * * *` | 每分钟的第 0 秒 |
| `0 0 * * * *` | 每小时整点 |
| `0 0 2 * * *` | 每天凌晨 2 点 |
| `0 0 2 * * 0` | 每周日凌晨 2 点 |
| `0 0 0 1 * *` | 每月 1 号零点 |
| `0 30 3 * * 1-5` | 工作日（周一到周五）凌晨 3:30 |

## 用法

```go
import "lychee-go/internal/cron"

// ---------- 添加函数型任务 ----------
cron.AddFunc("clean_logs", "0 0 2 * * *", func() error {
    logger.Info("开始清理日志...")
    // 实际清理逻辑
    return nil
})

// ---------- 启动调度器（由 boot.Boot() 调用） ----------
cron.Start()

// ---------- 手动操作 ----------
cron.Stop()                          // 停止调度（不会中断正在执行的任务）
count := cron.TaskCount()            // 获取任务数
cron.Remove("clean_logs")            // 移除某个任务
```

## 设计要点

- **单线程调度**：所有任务在一个 goroutine 中串行调度，避免竞态。
- **任务自行处理 panic**：如任务内可能 panic，建议在 `func() error` 中自行 `recover`。
- **时区**：使用服务器本地时区（建议服务器设置为 `Asia/Shanghai`）。
- **无持久化**：任务在进程内存中。进程重启时任务会重新注册（从代码中），但"上次运行时间"等状态会丢失。
